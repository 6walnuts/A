# 股票助手 & 量化研究

一个零构建、零框架的纯静态股票行情项目,包含两个互相独立的页面:

| 页面 | 给谁用 | 入口 |
|------|--------|------|
| **股票助手** | 父母(大字体、操作简单的 PWA) | `/`(index.html) |
| **量化研究** | 自己(只读行情研究工具) | `/quant.html` |

所有功能均为**只读行情查询**,不涉及任何交易、下单或账户操作,不构成投资建议。

## 功能

### 股票助手(index.html)

- **自选股**:A 股 + 美股实时报价,5 秒自动刷新,红涨绿跌
- **大盘**:上证、深证、创业板、道琼斯、纳斯达克、标普500
- **个股详情**:分时图、日K/周K/月K(lightweight-charts)
- **财经新闻**:新浪 7×24 快讯,每分钟自动刷新,焦点新闻红边条标识
- **添加**:按名称、代码、拼音搜索股票
- **到价提醒**:涨到/跌到某价时应用内响铃弹窗(仅应用打开时生效)
- **PWA**:可添加到 iPhone 主屏幕当 app 用,静态资源离线缓存

### 量化研究(quant.html)

- **市场看板**:六大指数、全市场涨幅/跌幅/成交额榜、行业板块涨幅榜,每分钟刷新
- **股票筛选**:全市场成交额前 1000 只,按涨跌幅/换手率/量比/市盈率/总市值组合筛选,表头排序
- **因子排名**:动量、换手、量比、低估值、小市值五因子 Z 分数加权,可调权重,输出 Top 50

## 项目结构

```
├── index.html      股票助手(界面 + 全部逻辑)
├── quant.html      量化研究(看板 / 筛选 / 因子排名)
├── api/hq.js       行情代理(Vercel Serverless Function,解决跨域和 GBK 乱码)
├── dev-server.py   本地开发服务器(模拟 Vercel:静态托管 + /api/hq,仅调试用)
├── manifest.json   PWA 配置
├── sw.js           Service Worker(静态资源离线缓存)
├── icon-*.png      图标
└── 部署教程.md      面向非程序员的完整部署与使用教程
```

## 数据来源(全部免费公开接口)

| 数据 | 来源 | 说明 |
|------|------|------|
| 实时报价 / 分时 / K线 | 腾讯行情(qt.gtimg.cn / ifzq.gtimg.cn) | A 股基本实时,美股约 15 分钟延迟 |
| 股票搜索 | 腾讯 smartbox | 支持名称/代码/拼音 |
| 财经新闻 | 新浪财经 7×24 直播 | |
| 全市场列表 / 板块 | 东方财富 push2 | 海外 IP 为延迟行情,单页上限 100 条(已做并行翻页) |

前端不直接访问上述接口,统一走 `/api/hq` 代理:`type=quote|minute|kline|search|news|emlist`,参数经白名单校验,只透传只读查询。

## 本地开发

无需 Node,系统自带 Python 3 即可:

```bash
python3 dev-server.py
# 打开 http://localhost:8787        股票助手
# 打开 http://localhost:8787/quant.html  量化研究
```

## 部署

推到 GitHub 后在 [Vercel](https://vercel.com) 导入本仓库,直接 Deploy 即可(无需任何构建配置)。之后每次 push,Vercel 自动重新部署。详细步骤(含教父母安装到 iPhone 主屏幕)见[部署教程.md](./部署教程.md)。

注意:`vercel.app` 域名在国内部分网络环境访问不稳定,必要时在 Vercel 里绑定自有域名。

## 已知限制

- 到价提醒仅在应用打开时生效(iOS 对网页应用的后台推送限制)
- 免费行情源无深度财务数据(ROE、营收等),量化研究页的估值因子仅有 PE/PB
- K线历史约数百根日K,适合技术面研究,不适合长周期精细回测
- 图表库 lightweight-charts 走 jsdelivr CDN,国内网络偶有加载失败
